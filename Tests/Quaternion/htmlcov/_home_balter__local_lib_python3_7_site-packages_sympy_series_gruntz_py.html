1""" 

2Limits 

3====== 

4 

5Implemented according to the PhD thesis 

6http://www.cybertester.com/data/gruntz.pdf, which contains very thorough 

7descriptions of the algorithm including many examples. We summarize here 

8the gist of it. 

9 

10All functions are sorted according to how rapidly varying they are at 

11infinity using the following rules. Any two functions f and g can be 

12compared using the properties of L: 

13 

14L=lim log|f(x)| / log|g(x)| (for x -> oo) 

15 

16We define >, < ~ according to:: 

17 

18 1. f > g .... L=+-oo 

19 

20 we say that: 

21 - f is greater than any power of g 

22 - f is more rapidly varying than g 

23 - f goes to infinity/zero faster than g 

24 

25 2. f < g .... L=0 

26 

27 we say that: 

28 - f is lower than any power of g 

29 

30 3. f ~ g .... L!=0, +-oo 

31 

32 we say that: 

33 - both f and g are bounded from above and below by suitable integral 

34 powers of the other 

35 

36Examples 

37======== 

38:: 

39 2 < x < exp(x) < exp(x**2) < exp(exp(x)) 

40 2 ~ 3 ~ -5 

41 x ~ x**2 ~ x**3 ~ 1/x ~ x**m ~ -x 

42 exp(x) ~ exp(-x) ~ exp(2x) ~ exp(x)**2 ~ exp(x+exp(-x)) 

43 f ~ 1/f 

44 

45So we can divide all the functions into comparability classes (x and x^2 

46belong to one class, exp(x) and exp(-x) belong to some other class). In 

47principle, we could compare any two functions, but in our algorithm, we 

48don't compare anything below the class 2~3~-5 (for example log(x) is 

49below this), so we set 2~3~-5 as the lowest comparability class. 

50 

51Given the function f, we find the list of most rapidly varying (mrv set) 

52subexpressions of it. This list belongs to the same comparability class. 

53Let's say it is {exp(x), exp(2x)}. Using the rule f ~ 1/f we find an 

54element "w" (either from the list or a new one) from the same 

55comparability class which goes to zero at infinity. In our example we 

56set w=exp(-x) (but we could also set w=exp(-2x) or w=exp(-3x) ...). We 

57rewrite the mrv set using w, in our case {1/w, 1/w^2}, and substitute it 

58into f. Then we expand f into a series in w:: 

59 

60 f = c0*w^e0 + c1*w^e1 + ... + O(w^en), where e0<e1<...<en, c0!=0 

61 

62but for x->oo, lim f = lim c0*w^e0, because all the other terms go to zero, 

63because w goes to zero faster than the ci and ei. So:: 

64 

65 for e0>0, lim f = 0 

66 for e0<0, lim f = +-oo (the sign depends on the sign of c0) 

67 for e0=0, lim f = lim c0 

68 

69We need to recursively compute limits at several places of the algorithm, but 

70as is shown in the PhD thesis, it always finishes. 

71 

72Important functions from the implementation: 

73 

74compare(a, b, x) compares "a" and "b" by computing the limit L. 

75mrv(e, x) returns list of most rapidly varying (mrv) subexpressions of "e" 

76rewrite(e, Omega, x, wsym) rewrites "e" in terms of w 

77leadterm(f, x) returns the lowest power term in the series of f 

78mrv_leadterm(e, x) returns the lead term (c0, e0) for e 

79limitinf(e, x) computes lim e (for x->oo) 

80limit(e, z, z0) computes any limit by converting it to the case x->oo 

81 

82All the functions are really simple and straightforward except 

83rewrite(), which is the most difficult/complex part of the algorithm. 

84When the algorithm fails, the bugs are usually in the series expansion 

85(i.e. in SymPy) or in rewrite. 

86 

87This code is almost exact rewrite of the Maple code inside the Gruntz 

88thesis. 

89

133def compare(a, b, x): 

134 """Returns "<" if a<b, "=" for a == b, ">" for a>b""" 

135 # log(exp(...)) must always be simplified here for termination 

136 la, lb = log(a), log(b) 

137 if isinstance(a, Basic) and isinstance(a, exp): 

138 la = a.args[0] 

139 if isinstance(b, Basic) and isinstance(b, exp): 

140 lb = b.args[0] 

141 

142 c = limitinf(la/lb, x) 

143 if c == 0: 

144 return "<" 

145 elif c.is_infinite: 

146 return ">" 

147 else: 

148 return "="

151class SubsSet(dict): 

152 """ 

153 Stores (expr, dummy) pairs, and how to rewrite expr-s. 

154 

155 The gruntz algorithm needs to rewrite certain expressions in term of a new 

156 variable w. We cannot use subs, because it is just too smart for us. For 

157 example:: 

158 

159 > Omega=[exp(exp(_p - exp(-_p))/(1 - 1/_p)), exp(exp(_p))] 

160 > O2=[exp(-exp(_p) + exp(-exp(-_p))*exp(_p)/(1 - 1/_p))/_w, 1/_w] 

161 > e = exp(exp(_p - exp(-_p))/(1 - 1/_p)) - exp(exp(_p)) 

162 > e.subs(Omega[0],O2[0]).subs(Omega[1],O2[1]) 

163 -1/w + exp(exp(p)*exp(-exp(-p))/(1 - 1/p)) 

164 

165 is really not what we want! 

166 

167 So we do it the hard way and keep track of all the things we potentially 

168 want to substitute by dummy variables. Consider the expression:: 

169 

170 exp(x - exp(-x)) + exp(x) + x. 

171 

172 The mrv set is {exp(x), exp(-x), exp(x - exp(-x))}. 

173 We introduce corresponding dummy variables d1, d2, d3 and rewrite:: 

174 

175 d3 + d1 + x. 

176 

177 This class first of all keeps track of the mapping expr->variable, i.e. 

178 will at this stage be a dictionary:: 

179 

180 {exp(x): d1, exp(-x): d2, exp(x - exp(-x)): d3}. 

181 

182 [It turns out to be more convenient this way round.] 

183 But sometimes expressions in the mrv set have other expressions from the 

184 mrv set as subexpressions, and we need to keep track of that as well. In 

185 this case, d3 is really exp(x - d2), so rewrites at this stage is:: 

186 

187 {d3: exp(x-d2)}. 

188 

189 The function rewrite uses all this information to correctly rewrite our 

190 expression in terms of w. In this case w can be chosen to be exp(-x), 

191 i.e. d2. The correct rewriting then is:: 

192 

193 exp(-w)/w + 1/w + x. 

194 """ 

195 def __init__(self): 

196 self.rewrites = {} 

197 

198 def __repr__(self): 

199 return super(SubsSet, self).__repr__() + ', ' + self.rewrites.__repr__() 

200 

201 def __getitem__(self, key): 

202 if not key in self: 

203 self[key] = Dummy() 

204 return dict.__getitem__(self, key) 

205 

206 def do_subs(self, e): 

207 """Substitute the variables with expressions""" 

208 for expr, var in self.items(): 

209 e = e.xreplace({var: expr}) 

210 return e 

211 

212 def meets(self, s2): 

213 """Tell whether or not self and s2 have non-empty intersection""" 

214 return set(self.keys()).intersection(list(s2.keys())) != set()

216 def union(self, s2, exps=None): 

217 """Compute the union of self and s2, adjusting exps""" 

218 res = self.copy() 

219 tr = {} 

220 for expr, var in s2.items(): 

221 if expr in self: 

222 if exps: 

223 exps = exps.xreplace({var: res[expr]}) 

224 tr[var] = res[expr] 

225 else: 

226 res[expr] = var 

227 for var, rewr in s2.rewrites.items(): 

228 res.rewrites[var] = rewr.xreplace(tr) 

229 return res, exps 

230 

231 def copy(self): 

232 """Create a shallow copy of SubsSet""" 

233 r = SubsSet() 

234 r.rewrites = self.rewrites.copy() 

235 for expr, var in self.items(): 

236 r[expr] = var 

237 return r

240@debug 

241def mrv(e, x): 

242 """Returns a SubsSet of most rapidly varying (mrv) subexpressions of 'e', 

243 and e rewritten in terms of these""" 

244 e = powsimp(e, deep=True, combine='exp') 

245 if not isinstance(e, Basic): 

246 raise TypeError("e should be an instance of Basic") 

247 if not e.has(x): 

248 return SubsSet(), e 

249 elif e == x: 

250 s = SubsSet() 

251 return s, s[x] 

252 elif e.is_Mul or e.is_Add: 

253 i, d = e.as_independent(x) # throw away x-independent terms 

254 if d.func != e.func: 

255 s, expr = mrv(d, x) 

256 return s, e.func(i, expr) 

257 a, b = d.as_two_terms() 

258 s1, e1 = mrv(a, x) 

259 s2, e2 = mrv(b, x) 

260 return mrv_max1(s1, s2, e.func(i, e1, e2), x) 

261 elif e.is_Pow: 

262 b, e = e.as_base_exp() 

263 if b == 1: 

264 return SubsSet(), b 

265 if e.has(x): 

266 return mrv(exp(e * log(b)), x) 

267 else: 

268 s, expr = mrv(b, x) 

269 return s, expr**e 

270 elif isinstance(e, log): 

271 s, expr = mrv(e.args[0], x) 

272 return s, log(expr) 

273 elif isinstance(e, exp): 

274 # We know from the theory of this algorithm that exp(log(...)) may always 

275 # be simplified here, and doing so is vital for termination. 

276 if isinstance(e.args[0], log): 

277 return mrv(e.args[0].args[0], x) 

278 # if a product has an infinite factor the result will be 

279 # infinite if there is no zero, otherwise NaN; here, we 

280 # consider the result infinite if any factor is infinite 

281 li = limitinf(e.args[0], x) 

282 if any(_.is_infinite for _ in Mul.make_args(li)): 

283 s1 = SubsSet() 

284 e1 = s1[e] 

285 s2, e2 = mrv(e.args[0], x) 

286 su = s1.union(s2)[0] 

287 su.rewrites[e1] = exp(e2) 

288 return mrv_max3(s1, e1, s2, exp(e2), su, e1, x) 

289 else: 

290 s, expr = mrv(e.args[0], x) 

291 return s, exp(expr) 

292 elif e.is_Function: 

293 l = [mrv(a, x) for a in e.args] 

294 l2 = [s for (s, _) in l if s != SubsSet()] 

295 if len(l2) != 1: 

296 # e.g. something like BesselJ(x, x) 

297 raise NotImplementedError("MRV set computation for functions in" 

298 " several variables not implemented.") 

299 s, ss = l2[0], SubsSet() 

300 args = [ss.do_subs(x[1]) for x in l] 

301 return s, e.func(*args) 

302 elif e.is_Derivative: 

303 raise NotImplementedError("MRV set computation for derviatives" 

304 " not implemented yet.") 

305 return mrv(e.args[0], x) 

306 raise NotImplementedError( 

307 "Don't know how to calculate the mrv of '%s'" % e)

310def mrv_max3(f, expsf, g, expsg, union, expsboth, x): 

311 """Computes the maximum of two sets of expressions f and g, which 

312 are in the same comparability class, i.e. max() compares (two elements of) 

313 f and g and returns either (f, expsf) [if f is larger], (g, expsg) 

314 [if g is larger] or (union, expsboth) [if f, g are of the same class]. 

315 """ 

316 if not isinstance(f, SubsSet): 

317 raise TypeError("f should be an instance of SubsSet") 

318 if not isinstance(g, SubsSet): 

319 raise TypeError("g should be an instance of SubsSet") 

320 if f == SubsSet(): 

321 return g, expsg 

322 elif g == SubsSet(): 

323 return f, expsf 

324 elif f.meets(g): 

325 return union, expsboth 

326 

327 c = compare(list(f.keys())[0], list(g.keys())[0], x) 

328 if c == ">": 

329 return f, expsf 

330 elif c == "<": 

331 return g, expsg 

332 else: 

333 if c != "=": 

334 raise ValueError("c should be =") 

335 return union, expsboth

338def mrv_max1(f, g, exps, x): 

339 """Computes the maximum of two sets of expressions f and g, which 

340 are in the same comparability class, i.e. mrv_max1() compares (two elements of) 

341 f and g and returns the set, which is in the higher comparability class 

342 of the union of both, if they have the same order of variation. 

343 Also returns exps, with the appropriate substitutions made. 

344 """ 

345 u, b = f.union(g, exps) 

346 return mrv_max3(f, g.do_subs(exps), g, f.do_subs(exps), 

347 u, b, x)

350@debug 

351@cacheit 

352@timeit 

353def sign(e, x): 

354 """ 

355 Returns a sign of an expression e(x) for x->oo. 

356 

357 :: 

358 

359 e > 0 for x sufficiently large ... 1 

360 e == 0 for x sufficiently large ... 0 

361 e < 0 for x sufficiently large ... -1 

362 

363 The result of this function is currently undefined if e changes sign 

364 arbitrarily often for arbitrarily large x (e.g. sin(x)). 

365 

366 Note that this returns zero only if e is *constantly* zero 

367 for x sufficiently large. [If e is constant, of course, this is just 

368 the same thing as the sign of e.] 

369 """ 

370 from sympy import sign as _sign 

371 if not isinstance(e, Basic): 

372 raise TypeError("e should be an instance of Basic") 

373 

374 if e.is_positive: 

375 return 1 

376 elif e.is_negative: 

377 return -1 

378 elif e.is_zero: 

379 return 0 

380 

381 elif not e.has(x): 

382 return _sign(e) 

383 elif e == x: 

384 return 1 

385 elif e.is_Mul: 

386 a, b = e.as_two_terms() 

387 sa = sign(a, x) 

388 if not sa: 

389 return 0 

390 return sa * sign(b, x) 

391 elif isinstance(e, exp): 

392 return 1 

393 elif e.is_Pow: 

394 s = sign(e.base, x) 

395 if s == 1: 

396 return 1 

397 if e.exp.is_Integer: 

398 return s**e.exp 

399 elif isinstance(e, log): 

400 return sign(e.args[0] - 1, x) 

401 

402 # if all else fails, do it the hard way 

403 c0, e0 = mrv_leadterm(e, x) 

404 return sign(c0, x)

407@debug 

408@timeit 

409@cacheit 

410def limitinf(e, x): 

411 """Limit e(x) for x-> oo""" 

412 # rewrite e in terms of tractable functions only 

413 e = e.rewrite('tractable', deep=True) 

414 

415 if not e.has(x): 

416 return e # e is a constant 

417 if e.has(Order): 

418 e = e.expand().removeO() 

419 if not x.is_positive: 

420 # We make sure that x.is_positive is True so we 

421 # get all the correct mathematical behavior from the expression. 

422 # We need a fresh variable. 

423 p = Dummy('p', positive=True, finite=True) 

424 e = e.subs(x, p) 

425 x = p 

426 c0, e0 = mrv_leadterm(e, x) 

427 sig = sign(e0, x) 

428 if sig == 1: 

429 return S.Zero # e0>0: lim f = 0 

430 elif sig == -1: # e0<0: lim f = +-oo (the sign depends on the sign of c0) 

431 if c0.match(I*Wild("a", exclude=[I])): 

432 return c0*oo 

433 s = sign(c0, x) 

434 # the leading term shouldn't be 0: 

435 if s == 0: 

436 raise ValueError("Leading term should not be 0") 

437 return s*oo 

438 elif sig == 0: 

439 return limitinf(c0, x) # e0=0: lim f = lim c0

476@debug 

477@timeit 

478@cacheit 

479def mrv_leadterm(e, x): 

480 """Returns (c0, e0) for e.""" 

481 Omega = SubsSet() 

482 if not e.has(x): 

483 return (e, S.Zero) 

484 if Omega == SubsSet(): 

485 Omega, exps = mrv(e, x) 

486 if not Omega: 

487 # e really does not depend on x after simplification 

488 series = calculate_series(e, x) 

489 c0, e0 = series.leadterm(x) 

490 if e0 != 0: 

491 raise ValueError("e0 should be 0") 

492 return c0, e0 

493 if x in Omega: 

494 # move the whole omega up (exponentiate each term): 

495 Omega_up = moveup2(Omega, x) 

496 e_up = moveup([e], x)[0] 

497 exps_up = moveup([exps], x)[0] 

498 # NOTE: there is no need to move this down! 

499 e = e_up 

500 Omega = Omega_up 

501 exps = exps_up 

502 # 

503 # The positive dummy, w, is used here so log(w*2) etc. will expand; 

504 # a unique dummy is needed in this algorithm 

505 # 

506 # For limits of complex functions, the algorithm would have to be 

507 # improved, or just find limits of Re and Im components separately. 

508 # 

509 w = Dummy("w", real=True, positive=True, finite=True) 

510 f, logw = rewrite(exps, Omega, x, w) 

511 series = calculate_series(f, w, logx=logw) 

512 return series.leadterm(w)

515def build_expression_tree(Omega, rewrites): 

516 r""" Helper function for rewrite. 

517 

518 We need to sort Omega (mrv set) so that we replace an expression before 

519 we replace any expression in terms of which it has to be rewritten:: 

520 

521 e1 ---> e2 ---> e3 

522 \ 

523 -> e4 

524 

525 Here we can do e1, e2, e3, e4 or e1, e2, e4, e3. 

526 To do this we assemble the nodes into a tree, and sort them by height. 

527 

528 This function builds the tree, rewrites then sorts the nodes. 

529 """ 

530 class Node: 

531 def ht(self): 

532 return reduce(lambda x, y: x + y, 

533 [x.ht() for x in self.before], 1) 

534 nodes = {} 

535 for expr, v in Omega: 

536 n = Node() 

537 n.before = [] 

538 n.var = v 

539 n.expr = expr 

540 nodes[v] = n 

541 for _, v in Omega: 

542 if v in rewrites: 

543 n = nodes[v] 

544 r = rewrites[v] 

545 for _, v2 in Omega: 

546 if r.has(v2): 

547 n.before.append(nodes[v2]) 

548 

549 return nodes

552@debug 

553@timeit 

554def rewrite(e, Omega, x, wsym): 

555 """e(x) ... the function 

556 Omega ... the mrv set 

557 wsym ... the symbol which is going to be used for w 

558 

559 Returns the rewritten e in terms of w and log(w). See test_rewrite1() 

560 for examples and correct results. 

561 """ 

562 from sympy import ilcm 

563 if not isinstance(Omega, SubsSet): 

564 raise TypeError("Omega should be an instance of SubsSet") 

565 if len(Omega) == 0: 

566 raise ValueError("Length can not be 0") 

567 # all items in Omega must be exponentials 

568 for t in Omega.keys(): 

569 if not isinstance(t, exp): 

570 raise ValueError("Value should be exp") 

571 rewrites = Omega.rewrites 

572 Omega = list(Omega.items()) 

573 

574 nodes = build_expression_tree(Omega, rewrites) 

575 Omega.sort(key=lambda x: nodes[x[1]].ht(), reverse=True) 

576 

577 # make sure we know the sign of each exp() term; after the loop, 

578 # g is going to be the "w" - the simplest one in the mrv set 

579 for g, _ in Omega: 

580 sig = sign(g.args[0], x) 

581 if sig != 1 and sig != -1: 

582 raise NotImplementedError('Result depends on the sign of %s' % sig) 

583 if sig == 1: 

584 wsym = 1/wsym # if g goes to oo, substitute 1/w 

585 # O2 is a list, which results by rewriting each item in Omega using "w" 

586 O2 = [] 

587 denominators = [] 

588 for f, var in Omega: 

589 c = limitinf(f.args[0]/g.args[0], x) 

590 if c.is_Rational: 

591 denominators.append(c.q) 

592 arg = f.args[0] 

593 if var in rewrites: 

594 if not isinstance(rewrites[var], exp): 

595 raise ValueError("Value should be exp") 

596 arg = rewrites[var].args[0] 

597 O2.append((var, exp((arg - c*g.args[0]).expand())*wsym**c)) 

598 

599 # Remember that Omega contains subexpressions of "e". So now we find 

600 # them in "e" and substitute them for our rewriting, stored in O2 

601 

602 # the following powsimp is necessary to automatically combine exponentials, 

603 # so that the .xreplace() below succeeds: 

604 # TODO this should not be necessary 

605 f = powsimp(e, deep=True, combine='exp') 

606 for a, b in O2: 

607 f = f.xreplace({a: b}) 

608 

609 for _, var in Omega: 

610 assert not f.has(var) 

611 

612 # finally compute the logarithm of w (logw). 

613 logw = g.args[0] 

614 if sig == 1: 

615 logw = -logw # log(w)->log(1/w)=-log(w) 

616 

617 # Some parts of sympy have difficulty computing series expansions with 

618 # non-integral exponents. The following heuristic improves the situation: 

619 exponent = reduce(ilcm, denominators, 1) 

620 f = f.xreplace({wsym: wsym**exponent}) 

621 logw /= exponent 

622 

623 return f, logw

626def gruntz(e, z, z0, dir="+"): 

627 """ 

628 Compute the limit of e(z) at the point z0 using the Gruntz algorithm. 

629 

630 z0 can be any expression, including oo and -oo. 

631 

632 For dir="+" (default) it calculates the limit from the right 

633 (z->z0+) and for dir="-" the limit from the left (z->z0-). For infinite z0 

634 (oo or -oo), the dir argument doesn't matter. 

635 

636 This algorithm is fully described in the module docstring in the gruntz.py 

637 file. It relies heavily on the series expansion. Most frequently, gruntz() 

638 is only used if the faster limit() function (which uses heuristics) fails. 

639 """ 

640 if not z.is_symbol: 

641 raise NotImplementedError("Second argument must be a Symbol") 

642 

643 # convert all limits to the limit z->oo; sign of z is handled in limitinf 

644 r = None 

645 if z0 == oo: 

646 r = limitinf(e, z) 

647 elif z0 == -oo: 

648 r = limitinf(e.subs(z, -z), z) 

649 else: 

650 if str(dir) == "-": 

651 e0 = e.subs(z, z0 - 1/z) 

652 elif str(dir) == "+": 

653 e0 = e.subs(z, z0 + 1/z) 

654 else: 

655 raise NotImplementedError("dir must be '+' or '-'") 

656 r = limitinf(e0, z) 

657 

658 # This is a bit of a heuristic for nice results... we always rewrite 

659 # tractable functions in terms of familiar intractable ones. 

660 # It might be nicer to rewrite the exactly to what they were initially, 

661 # but that would take some work to implement. 

662 return r.rewrite('intractable', deep=True)
