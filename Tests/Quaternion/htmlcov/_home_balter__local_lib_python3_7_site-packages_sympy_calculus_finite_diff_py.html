1""" 

2Finite difference weights 

3========================= 

4 

5This module implements an algorithm for efficient generation of finite 

6difference weights for ordinary differentials of functions for 

7derivatives from 0 (interpolation) up to arbitrary order. 

8 

9The core algorithm is provided in the finite difference weight generating 

10function (``finite_diff_weights``), and two convenience functions are provided 

11for: 

12 

13- estimating a derivative (or interpolate) directly from a series of points 

14 is also provided (``apply_finite_diff``). 

15- differentiating by using finite difference approximations 

16 (``differentiate_finite``). 

17 

18"""

25def finite_diff_weights(order, x_list, x0=S.One): 

26 """ 

27 Calculates the finite difference weights for an arbitrarily spaced 

28 one-dimensional grid (``x_list``) for derivatives at ``x0`` of order 

29 0, 1, ..., up to ``order`` using a recursive formula. Order of accuracy 

30 is at least ``len(x_list) - order``, if ``x_list`` is defined correctly. 

31 

32 

33 Parameters 

34 ========== 

35 

36 order: int 

37 Up to what derivative order weights should be calculated. 

38 0 corresponds to interpolation. 

39 x_list: sequence 

40 Sequence of (unique) values for the independent variable. 

41 It is useful (but not necessary) to order ``x_list`` from 

42 nearest to furthest from ``x0``; see examples below. 

43 x0: Number or Symbol 

44 Root or value of the independent variable for which the finite 

45 difference weights should be generated. Default is ``S.One``. 

46 

47 Returns 

48 ======= 

49 

50 list 

51 A list of sublists, each corresponding to coefficients for 

52 increasing derivative order, and each containing lists of 

53 coefficients for increasing subsets of x_list. 

54 

55 Examples 

56 ======== 

57 

58 >>> from sympy import S 

59 >>> from sympy.calculus import finite_diff_weights 

60 >>> res = finite_diff_weights(1, [-S(1)/2, S(1)/2, S(3)/2, S(5)/2], 0) 

61 >>> res 

62 [[[1, 0, 0, 0], 

63 [1/2, 1/2, 0, 0], 

64 [3/8, 3/4, -1/8, 0], 

65 [5/16, 15/16, -5/16, 1/16]], 

66 [[0, 0, 0, 0], 

67 [-1, 1, 0, 0], 

68 [-1, 1, 0, 0], 

69 [-23/24, 7/8, 1/8, -1/24]]] 

70 >>> res[0][-1] # FD weights for 0th derivative, using full x_list 

71 [5/16, 15/16, -5/16, 1/16] 

72 >>> res[1][-1] # FD weights for 1st derivative 

73 [-23/24, 7/8, 1/8, -1/24] 

74 >>> res[1][-2] # FD weights for 1st derivative, using x_list[:-1] 

75 [-1, 1, 0, 0] 

76 >>> res[1][-1][0] # FD weight for 1st deriv. for x_list[0] 

77 -23/24 

78 >>> res[1][-1][1] # FD weight for 1st deriv. for x_list[1], etc. 

79 7/8 

80 

81 Each sublist contains the most accurate formula at the end. 

82 Note, that in the above example ``res[1][1]`` is the same as ``res[1][2]``. 

83 Since res[1][2] has an order of accuracy of 

84 ``len(x_list[:3]) - order = 3 - 1 = 2``, the same is true for ``res[1][1]``! 

85 

86 >>> from sympy import S 

87 >>> from sympy.calculus import finite_diff_weights 

88 >>> res = finite_diff_weights(1, [S(0), S(1), -S(1), S(2), -S(2)], 0)[1] 

89 >>> res 

90 [[0, 0, 0, 0, 0], 

91 [-1, 1, 0, 0, 0], 

92 [0, 1/2, -1/2, 0, 0], 

93 [-1/2, 1, -1/3, -1/6, 0], 

94 [0, 2/3, -2/3, -1/12, 1/12]] 

95 >>> res[0] # no approximation possible, using x_list[0] only 

96 [0, 0, 0, 0, 0] 

97 >>> res[1] # classic forward step approximation 

98 [-1, 1, 0, 0, 0] 

99 >>> res[2] # classic centered approximation 

100 [0, 1/2, -1/2, 0, 0] 

101 >>> res[3:] # higher order approximations 

102 [[-1/2, 1, -1/3, -1/6, 0], [0, 2/3, -2/3, -1/12, 1/12]] 

103 

104 Let us compare this to a differently defined ``x_list``. Pay attention to 

105 ``foo[i][k]`` corresponding to the gridpoint defined by ``x_list[k]``. 

106 

107 >>> from sympy import S 

108 >>> from sympy.calculus import finite_diff_weights 

109 >>> foo = finite_diff_weights(1, [-S(2), -S(1), S(0), S(1), S(2)], 0)[1] 

110 >>> foo 

111 [[0, 0, 0, 0, 0], 

112 [-1, 1, 0, 0, 0], 

113 [1/2, -2, 3/2, 0, 0], 

114 [1/6, -1, 1/2, 1/3, 0], 

115 [1/12, -2/3, 0, 2/3, -1/12]] 

116 >>> foo[1] # not the same and of lower accuracy as res[1]! 

117 [-1, 1, 0, 0, 0] 

118 >>> foo[2] # classic double backward step approximation 

119 [1/2, -2, 3/2, 0, 0] 

120 >>> foo[4] # the same as res[4] 

121 [1/12, -2/3, 0, 2/3, -1/12] 

122 

123 Note that, unless you plan on using approximations based on subsets of 

124 ``x_list``, the order of gridpoints does not matter. 

125 

126 

127 The capability to generate weights at arbitrary points can be 

128 used e.g. to minimize Runge's phenomenon by using Chebyshev nodes: 

129 

130 >>> from sympy import cos, symbols, pi, simplify 

131 >>> from sympy.calculus import finite_diff_weights 

132 >>> N, (h, x) = 4, symbols('h x') 

133 >>> x_list = [x+h*cos(i*pi/(N)) for i in range(N,-1,-1)] # chebyshev nodes 

134 >>> print(x_list) 

135 [-h + x, -sqrt(2)*h/2 + x, x, sqrt(2)*h/2 + x, h + x] 

136 >>> mycoeffs = finite_diff_weights(1, x_list, 0)[1][4] 

137 >>> [simplify(c) for c in mycoeffs] #doctest: +NORMALIZE_WHITESPACE 

138 [(h**3/2 + h**2*x - 3*h*x**2 - 4*x**3)/h**4, 

139 (-sqrt(2)*h**3 - 4*h**2*x + 3*sqrt(2)*h*x**2 + 8*x**3)/h**4, 

140 6*x/h**2 - 8*x**3/h**4, 

141 (sqrt(2)*h**3 - 4*h**2*x - 3*sqrt(2)*h*x**2 + 8*x**3)/h**4, 

142 (-h**3/2 + h**2*x + 3*h*x**2 - 4*x**3)/h**4] 

143 

144 Notes 

145 ===== 

146 

147 If weights for a finite difference approximation of 3rd order 

148 derivative is wanted, weights for 0th, 1st and 2nd order are 

149 calculated "for free", so are formulae using subsets of ``x_list``. 

150 This is something one can take advantage of to save computational cost. 

151 Be aware that one should define ``x_list`` from nearest to farest from 

152 ``x0``. If not, subsets of ``x_list`` will yield poorer approximations, 

153 which might not grand an order of accuracy of ``len(x_list) - order``. 

154 

155 See also 

156 ======== 

157 

158 sympy.calculus.finite_diff.apply_finite_diff 

159 

160 

161 References 

162 ========== 

163 

164 .. [1] Generation of Finite Difference Formulas on Arbitrarily Spaced 

165 Grids, Bengt Fornberg; Mathematics of computation; 51; 184; 

166 (1988); 699-706; doi:10.1090/S0025-5718-1988-0935077-0 

167 

168 """ 

169 # The notation below closely corresponds to the one used in the paper. 

170 if order < 0: 

171 raise ValueError("Negative derivative order illegal.") 

172 if int(order) != order: 

173 raise ValueError("Non-integer order illegal") 

174 M = order 

175 N = len(x_list) - 1 

176 delta = [[[0 for nu in range(N+1)] for n in range(N+1)] for 

177 m in range(M+1)] 

178 delta[0][0][0] = S(1) 

179 c1 = S(1) 

180 for n in range(1, N+1): 

181 c2 = S(1) 

182 for nu in range(0, n): 

183 c3 = x_list[n]-x_list[nu] 

184 c2 = c2 * c3 

185 if n <= M: 

186 delta[n][n-1][nu] = 0 

187 for m in range(0, min(n, M)+1): 

188 delta[m][n][nu] = (x_list[n]-x0)*delta[m][n-1][nu] -\ 

189 m*delta[m-1][n-1][nu] 

190 delta[m][n][nu] /= c3 

191 for m in range(0, min(n, M)+1): 

192 delta[m][n][n] = c1/c2*(m*delta[m-1][n-1][n-1] - 

193 (x_list[n-1]-x0)*delta[m][n-1][n-1]) 

194 c1 = c2 

195 return delta

198def apply_finite_diff(order, x_list, y_list, x0=S(0)): 

199 """ 

200 Calculates the finite difference approximation of 

201 the derivative of requested order at ``x0`` from points 

202 provided in ``x_list`` and ``y_list``. 

203 

204 Parameters 

205 ========== 

206 

207 order: int 

208 order of derivative to approximate. 0 corresponds to interpolation. 

209 x_list: sequence 

210 Sequence of (unique) values for the independent variable. 

211 y_list: sequence 

212 The function value at corresponding values for the independent 

213 variable in x_list. 

214 x0: Number or Symbol 

215 At what value of the independent variable the derivative should be 

216 evaluated. Defaults to S(0). 

217 

218 Returns 

219 ======= 

220 

221 sympy.core.add.Add or sympy.core.numbers.Number 

222 The finite difference expression approximating the requested 

223 derivative order at ``x0``. 

224 

225 Examples 

226 ======== 

227 

228 >>> from sympy.calculus import apply_finite_diff 

229 >>> cube = lambda arg: (1.0*arg)**3 

230 >>> xlist = range(-3,3+1) 

231 >>> apply_finite_diff(2, xlist, map(cube, xlist), 2) - 12 # doctest: +SKIP 

232 -3.55271367880050e-15 

233 

234 we see that the example above only contain rounding errors. 

235 apply_finite_diff can also be used on more abstract objects: 

236 

237 >>> from sympy import IndexedBase, Idx 

238 >>> from sympy.calculus import apply_finite_diff 

239 >>> x, y = map(IndexedBase, 'xy') 

240 >>> i = Idx('i') 

241 >>> x_list, y_list = zip(*[(x[i+j], y[i+j]) for j in range(-1,2)]) 

242 >>> apply_finite_diff(1, x_list, y_list, x[i]) 

243 ((x[i + 1] - x[i])/(-x[i - 1] + x[i]) - 1)*y[i]/(x[i + 1] - x[i]) - \ 

244(x[i + 1] - x[i])*y[i - 1]/((x[i + 1] - x[i - 1])*(-x[i - 1] + x[i])) + \ 

245(-x[i - 1] + x[i])*y[i + 1]/((x[i + 1] - x[i - 1])*(x[i + 1] - x[i])) 

246 

247 Notes 

248 ===== 

249 

250 Order = 0 corresponds to interpolation. 

251 Only supply so many points you think makes sense 

252 to around x0 when extracting the derivative (the function 

253 need to be well behaved within that region). Also beware 

254 of Runge's phenomenon. 

255 

256 See also 

257 ======== 

258 

259 sympy.calculus.finite_diff.finite_diff_weights 

260 

261 References 

262 ========== 

263 

264 Fortran 90 implementation with Python interface for numerics: finitediff_ 

265 

266 .. _finitediff: https://github.com/bjodah/finitediff 

267 

268 """ 

269 

270 # In the original paper the following holds for the notation: 

271 # M = order 

272 # N = len(x_list) - 1 

273 

274 N = len(x_list) - 1 

275 if len(x_list) != len(y_list): 

276 raise ValueError("x_list and y_list not equal in length.") 

277 

278 delta = finite_diff_weights(order, x_list, x0) 

279 

280 derivative = 0 

281 for nu in range(0, len(x_list)): 

282 derivative += delta[order][N][nu]*y_list[nu] 

283 return derivative

286def _as_finite_diff(derivative, points=1, x0=None, wrt=None): 

287 """ 

288 Returns an approximation of a derivative of a function in 

289 the form of a finite difference formula. The expression is a 

290 weighted sum of the function at a number of discrete values of 

291 (one of) the independent variable(s). 

292 

293 Parameters 

294 ========== 

295 

296 derivative: a Derivative instance 

297 

298 points: sequence or coefficient, optional 

299 If sequence: discrete values (length >= order+1) of the 

300 independent variable used for generating the finite 

301 difference weights. 

302 If it is a coefficient, it will be used as the step-size 

303 for generating an equidistant sequence of length order+1 

304 centered around ``x0``. default: 1 (step-size 1) 

305 

306 x0: number or Symbol, optional 

307 the value of the independent variable (``wrt``) at which the 

308 derivative is to be approximated. Default: same as ``wrt``. 

309 

310 wrt: Symbol, optional 

311 "with respect to" the variable for which the (partial) 

312 derivative is to be approximated for. If not provided it 

313 is required that the Derivative is ordinary. Default: ``None``. 

314 

315 

316 Examples 

317 ======== 

318 

319 >>> from sympy import symbols, Function, exp, sqrt, Symbol, as_finite_diff 

320 >>> from sympy.utilities.exceptions import SymPyDeprecationWarning 

321 >>> import warnings 

322 >>> warnings.simplefilter("ignore", SymPyDeprecationWarning) 

323 >>> x, h = symbols('x h') 

324 >>> f = Function('f') 

325 >>> as_finite_diff(f(x).diff(x)) 

326 -f(x - 1/2) + f(x + 1/2) 

327 

328 The default step size and number of points are 1 and ``order + 1`` 

329 respectively. We can change the step size by passing a symbol 

330 as a parameter: 

331 

332 >>> as_finite_diff(f(x).diff(x), h) 

333 -f(-h/2 + x)/h + f(h/2 + x)/h 

334 

335 We can also specify the discretized values to be used in a sequence: 

336 

337 >>> as_finite_diff(f(x).diff(x), [x, x+h, x+2*h]) 

338 -3*f(x)/(2*h) + 2*f(h + x)/h - f(2*h + x)/(2*h) 

339 

340 The algorithm is not restricted to use equidistant spacing, nor 

341 do we need to make the approximation around ``x0``, but we can get 

342 an expression estimating the derivative at an offset: 

343 

344 >>> e, sq2 = exp(1), sqrt(2) 

345 >>> xl = [x-h, x+h, x+e*h] 

346 >>> as_finite_diff(f(x).diff(x, 1), xl, x+h*sq2) 

347 2*h*((h + sqrt(2)*h)/(2*h) - (-sqrt(2)*h + h)/(2*h))*f(E*h + x)/\ 

348((-h + E*h)*(h + E*h)) + (-(-sqrt(2)*h + h)/(2*h) - \ 

349(-sqrt(2)*h + E*h)/(2*h))*f(-h + x)/(h + E*h) + \ 

350(-(h + sqrt(2)*h)/(2*h) + (-sqrt(2)*h + E*h)/(2*h))*f(h + x)/(-h + E*h) 

351 

352 Partial derivatives are also supported: 

353 

354 >>> y = Symbol('y') 

355 >>> d2fdxdy=f(x,y).diff(x,y) 

356 >>> as_finite_diff(d2fdxdy, wrt=x) 

357 -Derivative(f(x - 1/2, y), y) + Derivative(f(x + 1/2, y), y) 

358 

359 See also 

360 ======== 

361 

362 sympy.calculus.finite_diff.apply_finite_diff 

363 sympy.calculus.finite_diff.finite_diff_weights 

364 

365 """ 

366 if derivative.is_Derivative: 

367 pass 

368 elif derivative.is_Atom: 

369 return derivative 

370 else: 

371 return derivative.fromiter( 

372 [_as_finite_diff(ar, points, x0, wrt) for ar 

373 in derivative.args], **derivative.assumptions0) 

374 

375 if wrt is None: 

376 old = None 

377 for v in derivative.variables: 

378 if old is v: 

379 continue 

380 derivative = _as_finite_diff(derivative, points, x0, v) 

381 old = v 

382 return derivative 

383 

384 order = derivative.variables.count(wrt) 

385 

386 if x0 is None: 

387 x0 = wrt 

388 

389 if not iterable(points): 

390 # points is simply the step-size, let's make it a 

391 # equidistant sequence centered around x0 

392 if order % 2 == 0: 

393 # even order => odd number of points, grid point included 

394 points = [x0 + points*i for i 

395 in range(-order//2, order//2 + 1)] 

396 else: 

397 # odd order => even number of points, half-way wrt grid point 

398 points = [x0 + points*S(i)/2 for i 

399 in range(-order, order + 1, 2)] 

400 others = [wrt, 0] 

401 for v in set(derivative.variables): 

402 if v == wrt: 

403 continue 

404 others += [v, derivative.variables.count(v)] 

405 if len(points) < order+1: 

406 raise ValueError("Too few points for order %d" % order) 

407 return apply_finite_diff(order, points, [ 

408 Derivative(derivative.expr.subs({wrt: x}), *others) for 

409 x in points], x0)

417def differentiate_finite(expr, *symbols, 

418 # points=1, x0=None, wrt=None, evaluate=True, #Py2: 

419 **kwargs): 

420 r""" Differentiate expr and replace Derivatives with finite differences. 

421 

422 Parameters 

423 ========== 

424 expr : expression 

425 \*symbols : differentiate with respect to symbols 

426 points: sequence or coefficient, optional 

427 see ``Derivative.as_finite_difference`` 

428 x0: number or Symbol, optional 

429 see ``Derivative.as_finite_difference`` 

430 wrt: Symbol, optional 

431 see ``Derivative.as_finite_difference`` 

432 evaluate : bool 

433 kwarg passed on to ``diff``, whether or not to 

434 evaluate the Derivative intermediately (default: ``False``). 

435 

436 

437 Examples 

438 ======== 

439 

440 >>> from sympy import cos, sin, Function, differentiate_finite 

441 >>> from sympy.abc import x, y, h 

442 >>> f, g = Function('f'), Function('g') 

443 >>> differentiate_finite(f(x)*g(x), x, points=[x-h, x+h]) 

444 -f(-h + x)*g(-h + x)/(2*h) + f(h + x)*g(h + x)/(2*h) 

445 

446 Note that the above form preserves the product rule in discrete form. 

447 If we want we can pass ``evaluate=True`` to get another form (which is 

448 usually not what we want): 

449 

450 >>> differentiate_finite(f(x)*g(x), x, points=[x-h, x+h], evaluate=True).simplify() 

451 -((f(-h + x) - f(h + x))*g(x) + (g(-h + x) - g(h + x))*f(x))/(2*h) 

452 

453 ``differentiate_finite`` works on any expression: 

454 

455 >>> differentiate_finite(f(x) + sin(x), x, 2) 

456 -2*f(x) + f(x - 1) + f(x + 1) - 2*sin(x) + sin(x - 1) + sin(x + 1) 

457 >>> differentiate_finite(f(x) + sin(x), x, 2, evaluate=True) 

458 -2*f(x) + f(x - 1) + f(x + 1) - sin(x) 

459 >>> differentiate_finite(f(x, y), x, y) 

460 f(x - 1/2, y - 1/2) - f(x - 1/2, y + 1/2) - f(x + 1/2, y - 1/2) + f(x + 1/2, y + 1/2) 

461 

462 """ 

463 # Key-word only arguments only available in Python 3 

464 points = kwargs.pop('points', 1) 

465 x0 = kwargs.pop('x0', None) 

466 wrt = kwargs.pop('wrt', None) 

467 evaluate = kwargs.pop('evaluate', False) 

468 if kwargs != {}: 

469 raise ValueError("Unknown kwargs: %s" % kwargs) 

470 

471 Dexpr = expr.diff(*symbols, evaluate=evaluate) 

472 return Dexpr.replace( 

473 lambda arg: arg.is_Derivative, 

474 lambda arg: arg.as_finite_difference(points=points, x0=x0, wrt=wrt))
